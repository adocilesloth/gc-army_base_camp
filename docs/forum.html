abc_forum

Class that controls the creation and archiving of campaign forums. Forums can only be created. This prevents Generals deleteing their army forums instead of leaving them to be archived.

Functions

  • start_forum
  • finish_forum
  • find_forums
  • add_forum
  • create_forum

start_forum

Creates the common forums: Campaign category, Tournament Administrator forum, Tournament Administrator + High Command forum, Results, Rules and Information forum. Assigns correct permissions for these forums. Move the category to be the second category.

Calls:
  • core/abc_forum->create_forum
  • include/abc_forum_inc->validate_range
Called by:
  • controller/main->handle

finish_forum

Archives the forums and sets permissions. The Tournament Administrator and Tournament Administrator + High Command forums are archived into a hidden forum so only Execs can see them.

Calls:
  • include/abc_sql_clean->sql_abc_clean
Called by:
  • controller/main->handle

find_forums

Searches for forums that a specific army can access. Also creates the html for adding new army forums.

Called by:
  • controller/main->handle

add_forum

Adds an army forum and moves it to be below the most recently made forum for that army.

Calls:
  • core/abc_forum->create_forum
  • include/abc_forum_inc->validate_range
  • include/abc_sql_clean->sql_abc_clean
Called by:
  • controller/main->handle

create_forum

Creates a forum.
Adapted from this phpBB forum post
Origional Author: Michael Fairchild

Arguments:
  • options - Dict of name, parent_id, forum_type (dict, optional)
  • forum_data - Data for the forum (see code) (dict, optional)
  • permissions - phpBB group permissions for the forum (see code) (dict, optional)
Returns:
  • forum_id - phpBB forum_id (int)
  • False - On failure (bool)
Calls:
  • include/abc_forum_inc->validate_range
Called by:
  • core/abc_forum->start_forum
  • core/abc_forum->add_forum
